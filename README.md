# OnlineSneakerStore

Учебный интернет-магазин кроссовок на Django 5 + SQLite с современным UI, личным кабинетом пользователя и админ-панелью магазина.

## Что реализовано

- Каталог кроссовок с карточками товаров, выбором размера и страницей товара
- Добавление в корзину без перезагрузки страницы (AJAX)
- Корзина с изменением количества, удалением и автопересчетом суммы
- Оформление заказа (имя, телефон, адрес)
- Регистрация и авторизация пользователей
- Личный кабинет пользователя:
  - редактирование профиля (в т.ч. аватар)
  - смена пароля
  - список своих заказов
  - отмена заказа в первые 10 минут
  - статус/подсказка по доставке
- Разделение ролей:
  - клиент
  - администратор (is_staff / is_superuser)
- Админ-панель магазина:
  - Dashboard
  - Заказы (поиск, фильтры, смена статуса)
  - Товары (добавление/редактирование/удаление)
  - Пользователи с заказами (имя клиента и телефон)
- Поддержка нескольких изображений у товара

## Технологии

- Python 3.11+
- Django 5.x
- SQLite
- HTML, CSS, JavaScript (fetch/AJAX)

## Быстрый запуск

1. Клонировать проект:

```bash
git clone https://github.com/k1llushka/OnlineSneakerStore.git
cd OnlineSneakerStore
```

2. Создать и активировать виртуальное окружение:

Windows (PowerShell):

```powershell
python -m venv venv
.\venv\Scripts\Activate.ps1
```

3. Установить зависимости:

```bash
pip install -r requirements.txt
```

4. Применить миграции:

```bash
python manage.py migrate
```

5. Создать администратора:

```bash
python manage.py createsuperuser
```

6. Запустить сервер:

```bash
python manage.py runserver
```

## Основные URL

- Витрина: `http://127.0.0.1:8000/`
- Вход: `http://127.0.0.1:8000/login/`
- Регистрация: `http://127.0.0.1:8000/register/`
- Корзина: `http://127.0.0.1:8000/cart/`
- Мои заказы: `http://127.0.0.1:8000/my-orders/`
- Профиль: `http://127.0.0.1:8000/account/`
- Админ-панель магазина: `http://127.0.0.1:8000/dashboard/`
- Django admin: `http://127.0.0.1:8000/admin/`

## Миграции, важное

Если возникает ошибка вида `no such table: store_userprofile`, значит не применены миграции.

Выполните:

```bash
python manage.py makemigrations
python manage.py migrate
```

## Структура (кратко)

- `store/models.py` — модели товаров, заказов и профиля пользователя
- `store/views.py` — витрина, корзина, заказы, аккаунт, dashboard
- `store/urls.py` — маршруты приложения
- `templates/` — HTML-шаблоны витрины, аккаунта и админ-панели
- `static/css/styles.css` — стили

## Как заказчику управлять веб-приложением

### Роль клиента

1. Зарегистрироваться на `/register/` и войти через `/login/`.
2. В каталоге выбрать товар и размер, нажать `Добавить в корзину`.
3. Открыть корзину (`/cart/`), проверить количество/сумму, при необходимости удалить позиции.
4. Оформить заказ на `/checkout/` (имя, телефон, адрес).
5. Отслеживать заказ в разделе `Мои заказы` (`/my-orders/`):
   - видно статус заказа;
   - показывается ориентировочная доставка;
   - доступна отмена в первые 10 минут.
6. Редактировать профиль на `/account/`:
   - имя пользователя, email, аватар;
   - смена пароля (с подтверждением текущего).

### Роль администратора

1. Войти под админ-аккаунтом через `/login/`.
2. Перейти в панель `/dashboard/`.
3. Раздел `Заказы` (`/dashboard/orders/`):
   - поиск по имени/телефону;
   - фильтр по статусу;
   - смена статуса заказа (`новый`, `в обработке`, `отправлен`, `доставлен`).
4. Раздел `Товары` (`/dashboard/products/`):
   - добавить товар (название, цена, описание, размеры, изображения);
   - редактировать товар;
   - удалить товар.
5. Раздел `Пользователи` (`/dashboard/users/`):
   - список пользователей, которые заказывали;
   - имя клиента и телефон из заказа;
   - количество заказов и сумма.
6. Для системного администрирования использовать Django admin `/admin/`.

### Операционный минимум (ежедневно)

1. Проверить новые заказы в `/dashboard/orders/`.
2. Проставить актуальные статусы заказов.
3. Проверить остатки и актуальность карточек в `/dashboard/products/`.
4. Проверить корректность контактов клиентов в `/dashboard/users/` перед созвоном/доставкой.
5. Контролировать, что сайт открывается и корзина оформляет заказы без ошибок.
